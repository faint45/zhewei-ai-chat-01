# 通用執行引擎架構表

## 階段與組件對照

| 階段 | 引擎組件 | 軟體 | 動作 | 輸出／權限 |
|------|----------|------|------|------------|
| **1–3** | 理解與安排邏輯 | 大腦 | Gemini 1.5 Pro | 讀取 D 槽規範檔 | 讀 |
| **4** | 執行產出核心 | 核心 | Claude 3.5 Sonnet | 寫入 `D:\Project\{任務}\Draft` | 寫 Draft |

| **8** | 完成歸檔節點 | 歸檔 | Python (Docker) | 搬移至 `D:\Project\{任務}\Done` | 寫 Done |

---

## 路徑約定

- **Draft**：`D:\Project\{任務}\Draft` — 初稿產出
- **Fix**：`D:\Project\{任務}\Fix` — 回饋修正後覆蓋
- **Done**：`D:\Project\{任務}\Done` — 通過最終門檻後歸檔

---

## 築未科技對應（可選對齊）

若與現有 D/Z 分流對齊，可將 `{任務}` 對應到專案或任務 ID，並約定：

| 通用路徑 | 築未對應說明 |
|----------|----------------------|
| `D:\Project\{任務}\Draft` | 熱資料產出，可對應 `D:\brain_workspace\processed\` 下任務子目錄或專案 Draft 目錄 |
| `D:\Project\{任務}\Fix` | 同上 D 槽，修正覆蓋用目錄 |
| `D:\Project\{任務}\Done` | 通過門檻後可搬至 `Z:\Zhewei_Brain\Reports\` 或專案 Done 目錄，符合「冷資料／長期記憶」分流 |

規範檔讀取仍以 **D 槽** 為準；歸檔節點（Python/Docker）搬移至 Done 後，若需長期留存可再同步至 Z 槽。
